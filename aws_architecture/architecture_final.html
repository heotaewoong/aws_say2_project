1-B DNS 라우팅 + 초기 데이터 이전
Route 53 + DataSync
Route 53 — DNS 전화번호부
사용자가 rare-link.ai 주소를 입력하면 Route 53이 "이 주소는 CloudFront 여기로 가세요"라고 알려줍니다.
인터넷의 전화번호부 역할. 도메인 → IP 주소 변환.
DataSync — 초기 배치 이전 전용
기존 병원 서버에 있던 X-ray 이미지들을 처음 한 번 S3로 대량 이전할 때만 사용합니다.
평소 진단 요청에는 사용 안 함. 초기 세팅 도구.
1-C VPC — 우리만의 가상 사설 네트워크
Amazon VPC

VPC(Virtual Private Cloud)는 AWS 안에 우리만 쓰는 격리된 네트워크 공간입니다.
마치 회사 내부 인트라넷처럼, 외부에서 직접 접근할 수 없고 우리가 허용한 경로로만 통신합니다.

🌐 Public Subnet (AZ1 + AZ2)
인터넷과 직접 연결되는 구역.
NAT Gateway가 여기 있음.
Private Subnet이 외부 인터넷에 나갈 때 이 NAT GW를 거침.
AZ1, AZ2 두 곳에 각각 배치 → 한 곳 장애 시 다른 곳으로 자동 전환.
🔒 Private Subnet
인터넷에서 직접 접근 불가능한 안전한 구역.
Lambda, SageMaker, ElastiCache, RDS가 여기 있음.
외부 API 호출 시 NAT GW를 통해서만 나갈 수 있음.
S3·DynamoDB·Bedrock은 VPC Endpoint로 인터넷 없이 접근.
🔗 VPC Endpoint
Private Subnet → S3, DynamoDB, Bedrock 접근 시 인터넷을 거치지 않고 AWS 내부망으로 직접 연결.
Gateway Endpoint: S3, DynamoDB (무료)
Interface Endpoint: Bedrock, SageMaker (PrivateLink, 유료)
보안 강화 + NAT GW 비용 절감.
💡 NAT Gateway가 뭔가요: Private Subnet 안의 Lambda가 PubMed API 같은 외부 인터넷에 접근할 때 쓰는 출구입니다. 외부에서 안으로는 들어올 수 없고, 안에서 밖으로만 나갈 수 있어요. AZ1, AZ2 각각 하나씩 두어서 한 곳이 죽어도 서비스가 계속됩니다.
2 보안 관문 통과 — WAF → API Gateway → Cognito → Step Functions
VPC 외부 Regional Services
🛡️ WAF (Web Application Firewall)
CloudFront 앞(Global)과 API Gateway 앞(Regional) 두 곳에 배치.
SQL 인젝션, XSS, DDoS 공격 패턴 자동 차단.
의료 시스템 해킹 시도 1차 방어선.
🔌 API Gateway
우리 시스템의 정문. 5개 엔드포인트 관리.
Rate Limiting: 초당 요청 수 제한 (과부하 방지).
FHIR R4 호환 → BESTCare 등 EMR 시스템과 바로 연동 가능.
VPC 외부에 있어서 인터넷에서 직접 접근 가능.
🔑 Cognito — 로그인 인증
의사 계정 로그인 처리. JWT 토큰 발급.
"MD role" 확인 → 의사만 진단 API 호출 가능.
API Gateway Authorizer로 연결 → 토큰 없으면 요청 차단.
⚙️ Step Functions — 파이프라인 지휘자
API Gateway에서 직접 통합(Service Integration)으로 호출.
Phase 1A + 1B + 1C를 병렬(Parallel State)로 동시 실행.
3개 에이전트 모두 완료되면 Phase 2로 넘김.
VPC 외부 Regional Service → Lambda들은 VPC 내부에서 실행.
💡 Step Functions가 뭔가요: 여러 Lambda를 순서대로 또는 동시에 실행하는 워크플로우 관리자입니다. "X-ray 분석, 텍스트 분석, 혈액검사 분석을 동시에 시작하고, 셋 다 끝나면 다음 단계로 가라"는 로직을 코드 없이 상태 머신으로 정의합니다.
3 Lambda가 파이프라인 5단계 실행
Lambda (rag_pipeline.py)

Lambda는 서버 없이 코드만 실행하는 서비스입니다. rag_pipeline.py의 RareLinkPipeline 클래스가 여기서 실행됩니다.

① INPUT — X-ray는 SooNet(AI 모델)으로, 증상 텍스트는 Bedrock Haiku(Claude)로, 혈액검사는 규칙으로 HPO 코드 변환
② SCORE — Orphanet 4335개 희귀질환 데이터와 비교해서 가능성 높은 질환 순위 계산 (LIRICAL)
②-B SCORE — lung_disease_profiles.yaml 기반으로 일반 폐질환 Top10 랭킹 계산 (Rule-based)
③ TRIG — 희귀질환이거나 1/2위 차이가 작으면 RAG 심화 분석 실행
④ GEN — 외부 논문/케이스 검색 후 Claude Sonnet으로 소견서 생성
⑤ OUT — 결과를 DynamoDB에 저장하고 의사에게 반환
💡 왜 필요한가: Lambda는 요청이 없을 때 비용이 0원. 서버를 24시간 켜두지 않아도 됩니다.
3-B 캐시 · 큐 · 알림 — ElastiCache, SQS, SNS
Private Subnet
🗄️ ElastiCache (Redis) — HPO 캐시 버퍼
Phase 1A + 1B + 1C가 각각 계산한 HPO 코드를 임시 저장하는 버퍼.
Step Functions가 3개 에이전트 완료를 확인하고 Redis에서 결과를 읽어 Phase 2로 전달.
같은 HPO 조합이 반복되면 API 재호출 없이 캐시에서 바로 반환 → 속도 향상.
주의: 동기화 주체는 Step Functions. Redis는 캐싱/버퍼 목적만.
📬 SQS — 비동기 요청 큐
진단 요청이 한꺼번에 몰릴 때 Lambda 타임아웃(900초) 위험 방지.
요청을 큐에 쌓아두고 Lambda가 순서대로 처리.
Dead Letter Queue(DLQ): 처리 실패한 요청을 별도 큐에 보관 → 나중에 재처리 가능.
MLOps 트리거 버퍼로도 활용 (EventBridge → SQS → SageMaker Training).
🔔 SNS — 알림 발송
CloudWatch Alarm이 이상 감지 → SNS Topic → 이메일/Slack 자동 발송.
알림 예시:
• Lambda 에러율 >5% → 즉시 알림
• SageMaker 응답 지연 >30초 → 즉시 알림
• 모델 재학습 완료 → 완료 알림
• DynamoDB 쓰기 실패 → 즉시 알림
3-C 벡터 지식베이스 — RDS PostgreSQL (pgvector)
Private Subnet · Multi-AZ

Orphanet에 등록된 희귀질환 528개의 설명을 AI가 숫자 벡터로 변환해서 저장해둔 데이터베이스입니다.
RAG 단계에서 환자 HPO 코드를 벡터로 변환 → 가장 유사한 희귀질환을 의미 기반으로 검색합니다.

pgvector가 뭔가요?
PostgreSQL에 벡터 검색 기능을 추가하는 확장 플러그인.
"이 증상 조합과 가장 비슷한 질환이 뭔가요?"를 수학적으로 계산.
OpenSearch(비싼 검색 서비스) 대비 비용 1/10.
Multi-AZ — 자동 장애 복구
Primary DB (AZ1) → Standby DB (AZ2) 실시간 동기 복제.
AZ1 서버가 죽으면 60~120초 내 AZ2로 자동 전환.
의사가 진단 중에 DB 서버가 죽어도 서비스 계속.
Read Replica — 읽기 부하 분산
Primary: 쓰기 전용 (새 질환 임베딩 추가).
Read Replica: 읽기 전용 (RAG 벡터 검색).
동시에 여러 의사가 검색해도 Primary에 부하 없음.
4 X-ray 분석 — SageMaker
SageMaker Endpoint

X-ray 이미지는 SageMaker Endpoint에 배포된 SooNet 모델이 분석합니다.
SooNet은 DenseNet-121 기반 AI로, 14가지 폐 소견(폐렴, 흉막삼출, 기흉 등)의 확률값을 반환합니다.
각 소견은 HPO 코드로 변환되어 다음 단계로 전달됩니다.

💡 왜 SageMaker인가: GPU가 필요한 딥러닝 추론을 Lambda에서 직접 실행하면 메모리 초과. SageMaker는 GPU 서버를 별도로 관리해줍니다.
5 HPO 코드 변환 — Bedrock Haiku
Bedrock Claude Haiku

의사가 쓴 소견서 텍스트("호흡곤란과 흉통이 있습니다")를 Claude Haiku가 읽고
표준 HPO 코드(HP:0002094 = 호흡곤란, HP:0002202 = 흉막삼출)로 변환합니다.
Positive(있는 증상)와 Negative(없는 증상)를 분리해서 반환합니다.
Monarch Initiative API로 HPO 코드를 한국어 증상명으로 변환해 LLM 가독성을 높입니다.

💡 HPO가 뭔가요: Human Phenotype Ontology. 전 세계 의사들이 공통으로 쓰는 증상 코드 체계입니다. "호흡곤란"을 HP:0002094로 표준화해서 어떤 언어든 같은 의미로 처리 가능합니다.
6 질환 랭킹 계산 — LIRICAL + Rule-based
Lambda (lirical_scorer.py)

두 가지 방식으로 질환 랭킹을 계산합니다:

희귀질환 LIRICAL
Orphanet 4335개 희귀질환 DB와 환자 HPO를 비교.
각 질환에서 이 증상이 얼마나 자주 나타나는지 통계로 LR 점수 계산.
Top 10 희귀질환 랭킹 반환.
일반 질환 Rule-based
lung_disease_profiles_v2.yaml 기반.
HPO + X-ray 소견 + Lab 수치를 가중치 합산.
COPD, 폐렴 등 일반 폐질환 Top 10 랭킹 반환.
💡 LR이 뭔가요: Likelihood Ratio(가능도비). "이 증상 조합이 이 질환에서 얼마나 자주 나타나는가"를 수치화한 것. 높을수록 그 질환일 가능성이 높습니다.
7 RAG 트리거 판단
step3_rag_trigger()

두 조건 중 하나라도 해당하면 RAG(심화 분석)를 실행합니다:

조건 1 — 1위 질환이 희귀질환인 경우
(Orphanet에 등록된 질환 = 희귀질환)
조건 2 — 1위/2위 LR 점수 비율이 3.0 미만인 경우
(점수 차이가 작으면 불확실 → 더 조사 필요)
💡 RAG가 뭔가요: Retrieval-Augmented Generation. AI가 답변을 만들 때 외부 자료를 실시간으로 검색해서 근거로 사용하는 방식입니다.
8 외부 API 검색 — 4개 소스
NAT Gateway 경유

RAG가 실행되면 4개 외부 API에서 근거 자료를 수집합니다:

PubCaseFinder (일본 DBCLS)
HPO 코드 → 유사 희귀질환 케이스 + 관련 유전자
서버 장애 시 로컬 Orphanet CSV로 자동 폴백
PubMed E-utilities (NIH)
1위 질환명 → 최신 논문 3편 (PMID, 제목, Abstract)
PMID 실존 여부 자동 검증 (환각 방지)
ClinicalTrials.gov (NIH)
1위 질환명 → 현재 모집 중인 임상시험 3건
NCT ID, 단계(Phase), 기관, 요약문 포함
Monarch Initiative (EMBL-EBI)
HPO 코드 → 한국어/영어 증상명 변환
로컬 폴백 사전 30개 내장 (API 실패 시 대비)
💡 NAT Gateway가 뭔가요: VPC 내부(프라이빗 서브넷)에 있는 Lambda가 외부 인터넷(PubMed 등)에 접근할 때 거치는 출구입니다. 보안을 유지하면서 외부 통신 가능.
9 최종 소견서 생성 — Claude Sonnet
Bedrock Claude Sonnet 3.5

수집된 모든 데이터를 Claude Sonnet 3.5에게 전달해서 최종 진단 보조 리포트를 생성합니다.

LLM 인풋 (유저 프롬프트):
A. Symptom Raw (소견서 원문) + X-ray Top10 확률값 + Lab Raw 수치
B. HPO 코드 + 증상명 (Positive/Negative)
C. 희귀질환 LIRICAL 랭킹 Top5 + 일반 질환 Rule-based 랭킹 Top10
D. API 검색 결과 (PubCaseFinder + PubMed + ClinicalTrials)

출력 형식: JSON (구조화 데이터) + Markdown (의사용 소견서) 이중 출력
파라미터: temperature=0.2 (낮은 창의성, 일관성 중시), max_tokens=4000
💡 왜 VPC Endpoint인가: Lambda → Bedrock 통신이 인터넷을 거치지 않고 AWS 내부 네트워크로만 이동. 보안 강화 + 속도 향상.
10 결과 저장 — DynamoDB
DynamoDB

생성된 리포트와 진단 이력을 DynamoDB에 저장합니다.
두 가지 테이블을 사용합니다:

diagnosis-history
환자별 진단 이력 저장
PK: 환자 MRN, SK: 진단 시각
rare-case-collection
희귀 케이스 축적 (재학습용)
100건 쌓이면 자동 재학습 트리거
💡 왜 DynamoDB인가: RDS(관계형 DB)보다 빠르고 서버리스. 환자 수가 늘어도 자동으로 확장됩니다.
11 모델 자동 재학습 — MLOps
EventBridge → SageMaker

새로운 희귀 케이스가 100건 이상 쌓이면 AI가 스스로 재학습합니다:

① DynamoDB rare-case-collection에 케이스 축적
② EventBridge가 100건 도달 감지 → SageMaker Training Job 자동 실행
③ 새 데이터 + 기존 데이터로 SooNet 모델 재학습
④ S3에 새 모델 가중치 저장
⑤ SageMaker Endpoint Blue/Green 배포 — 다운타임 0초
💡 Blue/Green 배포가 뭔가요: 기존 모델(Blue)을 켜둔 채로 새 모델(Green)을 준비. 준비 완료되면 트래픽을 Green으로 전환. 서비스 중단 없이 모델 교체 가능.
11-B 컨테이너 이미지 저장소 — Amazon ECR
Elastic Container Registry

SageMaker Training Job이 SooNet 모델을 학습할 때 필요한 Python 환경(코드 + 라이브러리)을 Docker 이미지로 패키징해서 저장하는 곳입니다.

Docker 이미지가 뭔가요?
"이 코드를 실행하려면 Python 3.10, PyTorch 2.0, numpy 1.24가 필요합니다"를 하나의 패키지로 묶은 것.
어떤 서버에서 실행해도 동일한 환경 보장.
SageMaker Training Job이 ECR에서 이미지를 가져와서 학습 실행.
Inspector와 연동
Amazon Inspector가 ECR 이미지를 자동 스캔.
PyTorch, numpy 등 의존성 패키지의 CVE 취약점 탐지.
취약점 발견 시 Security Hub에 자동 보고.
12 보안 체계 전체 — IAM · KMS · Secrets Manager · CloudTrail · GuardDuty · Detective · Inspector · Security Hub
🔐 IAM — 권한 관리
"Lambda는 S3 읽기만 가능, 쓰기는 불가"처럼 최소 권한 원칙 적용.
각 서비스마다 필요한 권한만 부여.
역할(Role) 기반 → 키 없이 서비스 간 안전한 통신.
🗝️ KMS — 암호화 키 관리
S3에 저장된 X-ray 이미지, 모델 가중치를 암호화.
DynamoDB 환자 진단 이력 암호화.
KMS가 암호화 키를 안전하게 보관 → 키 없이는 데이터 읽기 불가.
HIPAA 의료 데이터 보호 필수 요건.
🔒 Secrets Manager — API 키 금고
Bedrock API 키, NCBI API Key를 코드에 직접 쓰지 않고 여기 저장.
Lambda가 실행될 때 런타임에 동적으로 가져옴.
키가 코드에 노출되면 GitHub에 올라갈 위험 → Secrets Manager로 완전 차단.
📜 CloudTrail — 감사 로그
"누가 언제 어떤 환자 데이터에 접근했는지" 모든 API 호출 기록.
S3에 로그 저장 → 90일 보관.
HIPAA 감사 요건 충족.
보안 사고 발생 시 "언제 무슨 일이 있었는지" 추적 가능.
🛡️ GuardDuty — 위협 탐지기
비정상 API 호출 패턴 감지.
예: "Lambda가 갑자기 대량의 S3 파일을 다운로드" → 자격증명 탈취 의심.
CloudTrail + VPC Flow Logs + DNS 로그를 ML로 분석.
탐지 즉시 Security Hub에 보고.
🔎 Detective — 침해 조사관
GuardDuty가 "이상하다"고 탐지하면, Detective가 "왜, 어떻게 일어났는지" 조사.
어떤 IAM 역할 → 어떤 API 호출 → 어떤 리소스 접근했는지 시각화.
의료 데이터 침해 사고 원인 파악에 필수.
🔬 Inspector — 취약점 스캐너
Lambda 함수의 Python 패키지 CVE 취약점 자동 스캔.
ECR 이미지(SageMaker 학습용 Docker) 취약점 탐지.
"boto3 1.26.0에 보안 취약점 있음 → 업그레이드 필요" 자동 알림.
